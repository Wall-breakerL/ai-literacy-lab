# 场景设计思考记录

> 生成日期：2026-04-07
> 背景：Human-AI Performance Lab 2-3 个场景的组合设计讨论

---

## 一、FAA vs MBTI 的场景需求差异

| 类型 | 测量目标 | 场景特征 |
|------|---------|---------|
| **FAA** | 任务完成质量 | 有最优解/可评估结果 |
| **MBTI** | 协作习惯倾向 | 没有单一正确答案；过程即信号 |

**关键洞察**：
- FAA 靠**决策型任务**（租房/旅行）
- MBTI 靠**创造/表达/评审型任务**（没有对错，行为方式被暴露）

## 二、场景组合方案

### 最终推荐：3 场景组合

| 场景 | 类型 | 核心测量目标 |
|------|------|------------|
| **租房** | 决策型 | FAA 全维度（Frame/Edit/Synthesize 强） |
| **旅行规划** | 决策型 | FAA 全维度 + MBTI（Ask/Review/Epistemic 强） |
| **AI 教学 + 检测** | 创造/表达型 | MBTI 全维度（Epistemic 极强） |

### 各场景分析

#### 租房（当前已有）

| FAA 维度 | 覆盖度 | MBTI 区分度 |
|---------|--------|------------|
| Frame | ✅ 强（约束明确） | ✅ 强（价格优先 vs 位置优先） |
| Ask | ✅ 强（多维度可问） | ✅ 中（依赖型 vs 独立型） |
| Review | ✅ 中（需核实信息） | ✅ 强（信不信 AI 描述） |
| Edit | ✅ 强（约束调整） | ✅ 强（全局重评 vs 局部调整） |
| Synthesize | ✅ 强（理由评估） | ✅ 中 |

#### 旅行规划（待开发）

| FAA 维度 | 覆盖度 | MBTI 区分度 |
|---------|--------|------------|
| Frame | ✅ 强 | ✅ 强（先定预算 vs 先定目的地） |
| Ask | ✅ 强（交通/住宿/景点可组合） | ✅ 强（问深问浅差异大） |
| Review | ✅ 强（行程冲突、季节问题） | ✅ 强（容易产生信息质疑） |
| Edit | ✅ 强（改目的地、换交通） | ✅ 强 |
| Synthesize | ✅ 强 | ✅ 中 |

## 三、MBTI 专属场景：AI 教学 + 检测

### 为什么需要这个场景

AI 教学 + 检测相比其他候选场景的优势：

| 特点 | 说明 |
|------|------|
| **内置反馈循环** | 有对错之分，用户知道自己对不对 |
| **暴露脆弱性** | 学习意味着"有不懂的"，触发 Relation |
| **Epistemic 极化** | 考差了：有人怪 AI 教得不好，有人承认自己没理解 |
| **自然 Edit 触发** | 错了就要重学，天然带 Edit |
| **RepairScope 明显** | 重学全部 vs 只学错题 |

### 候选场景对比

| 场景 | Relation | Workflow | Epistemic | RepairScope | 结论 |
|------|---------|---------|-----------|------------|------|
| 故事共创 | ✅ | ✅ | ⚠️ | ✅ | 可用但缺反馈 |
| 学习请求 | ✅ | ✅ | ✅ | ✅ | 有潜力但太开放 |
| **AI教学+检测** | ✅ | ✅ | **✅✅** | ✅ | **最适合** |
| 广告语 | ✅ | ✅ | ⚠️ | ✅ | 缺客观反馈 |
| 方案评审 | ✅ | ✅ | ✅ | ✅ | 可用但和旅行重叠 |

### 任务流程

```
阶段1：AI 讲解一个概念
阶段2：AI 出检测题（3道）
阶段3：用户看结果，AI 分析
         ↓
观察点（按阶段）：

阶段1：
- 用户说"太简单了" → Instrumental
- 用户说"为什么是这个逻辑" → Collaborative
- 用户说"等等，让我消化一下" → Workflow（探索型）

阶段2：
- 用户认真作答 → Trusting
- 用户说"这题出得不对" → Auditing
- 用户乱答/跳过 → Instrumental

阶段3：
- 用户接受评价 → Trusting
- 用户质疑"你讲的时候没说这个" → Auditing
- 用户说"那我再学一遍" → Collaborative + Edit
- 用户说"这个知识点换一种方式讲" → Local-Refinement
- 用户说"整个重新来" → Global-Reframing
```

## 四、主题选择：方式 B（预设列表）

### 三种方式对比

| 方式 | 可行性 | 测试质量 | 开发成本 |
|------|--------|---------|---------|
| A：用户完全自由出题 | ❌ 不可控因素太多 | 差 | 低 |
| B：预设列表选择 | ✅ 推荐 | 高 | 中 |
| C：混合（用户说 → LLM生成） | ⚠️ 可行但不可比 | 中 | 低 |

### 推荐：5 个预设主题

满足条件：普通人能理解 / 有明确对错 / 能触发质疑行为

| 主题 | 检测点举例 |
|------|----------|
| 通货膨胀 | 通胀率上升时会发生什么 |
| K线基础 | 怎么看涨跌 |
| 疫苗原理 | 疫苗是怎么起作用的 |
| 机器学习入门 | 什么是训练数据 |
| 复利 | 10万年化5%十年变多少 |

### 降级处理

用户选择"其他（自己说）"时：
- LLM 临时生成内容 + 检测题
- 这一轮数据不参与正式评估（标记为探索性数据）

## 六、一句话结论

> **FAA 靠决策型任务（租房/旅行），MBTI 靠 AI 教学+检测（内置反馈，Epistemic 极化）。**
>
> **场景数量：2-3 个，核心是租房 + AI 教学+检测，旅行作为可选第三场景。**
