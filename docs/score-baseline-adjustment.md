# 得分基线调整方案（让分数相对更高、更有激励感）

目标：避免一上来就给很低分，让用户觉得「有参与就有基本分」，同时保留区分度。

---

## 可调环节

| 环节 | 作用 | 风险 |
|------|------|------|
| **1. 规则 Judge v2 映射** | 无事件/少事件时给的分数区间 | 低：只影响无 LLM 时的规则打分 |
| **2. 规则校正 v2 上限** | 无证据时维度封顶等 | 中：放宽后无证据也能显示更高 |
| **3. Judge v2 提示词** | 让 LLM 锚定合理区间 | 低：不改变规则，只影响模型 |
| **4. 展示层曲线** | 同一原始分显示得更高（如乘系数或平移） | 中：需区分「原始分」与「展示分」 |

---

## 推荐组合（与 v2 代码对齐时）

- **1. 规则 Judge**：改 `lib/judge-rule-v2.ts` 中事件计数 → 各维 `score` 的映射。
- **2. Judge 提示词锚定**：改 `lib/llm/judge-v2.ts` 中原则与锚定描述。
- **3. 规则校正**：改 `lib/rule-corrector-v2.ts`（如各维封顶、敏感组合扣分）。

不采用展示层曲线，以免和后续校准、导出数据混淆；若你希望「同样逻辑下界面分数再高一点」，可以再加一层展示系数。

---

## 若仍觉得低，可再加

- 在 **Experience 卡**或**结果页**增加「解释性文案」，强调「成长空间」而非单纯分数比较。
- 离线脚本 `scripts/calibrate.mjs` 聚合体验卡后，人工标注再回调 `judge-rule-v2` / 提示词。
