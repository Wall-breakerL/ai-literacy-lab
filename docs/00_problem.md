# 项目定位与核心研究问题

## 项目定位

本项目不叫「Prompt 打分器」，而叫：**AI 交互素养评估原型**。

核心目标：在自然、轻松的任务对话中，评估用户与 Agent 协作解决问题的能力（有效、批判、负责地参与 AI 系统）。

## 研究问题

1. AI 素养能否被操作化为「自然对话中的 Agent 协作能力」？
2. 能否通过「身份 + 叙事场景（蓝图）」提升评估的真实感，同时保持统一的核心 rubric？

## 两天范围（MVP 边界）

- 身份由 `/setup` 的 IdentityDossier 注入；**场景**以 **v2 蓝图**为主（见 `docs/09`），不再维护旧四 JSON 场景。
- 核心 rubric 以 **v2 两层七维**为准（见 `docs/10`）；与旧「五维」文档的关系见 `docs/03` 文首说明。
- 交付：可运行蓝图对话 + v2 事件 + Judge v2 + 规则校正 v2 + 结果页展示。
