# AI-HQ Phase 2 测量设计

## 维度总览

| 顺序 | 维度 | 对应 Harness 层 | 测量轮次 | 总分 |
|-----|------|--------------|---------|-----|
| 1 | Route | L2 工具系统层 | 1轮 | 30 |
| 2 | Frame | L1 信息边界层 | 2轮 | 30 |
| 3 | Workflow | L3+L5 执行编排+评估观测 | 1轮 | 20 |
| 4 | Repair | L6 约束、校验与恢复层 | 1轮 | 20 |
| | **合计** | | | **100** |

> State（L4）取消独立维度，并入 Workflow 第4个探针。

---

## Route（执行配置）｜30分

**定义**：用户是否理解 Agent 的能力边界（LLM + Tools + Memory），知道什么时候需要外部工具介入、记忆管理，以及什么时候需要人工介入。

**测量：一轮开放问题**

> 「你觉得 Agent 是什么？它和 LLM（大语言模型，例如 GPT）有什么区别？平时你用 AI 完成任务之后，会自己再判断一下结果，还是直接使用？」

**探针（提到即得分）：**
- [ ] 提到 Agent 可以调用外部工具/连接外部世界（MCP 方向）：**10分**
- [ ] 提到 Agent 有记忆/状态管理能力：**10分**
- [ ] 提到需要人工判断/不会直接使用 AI 输出：**10分**

---

## Frame（任务契约）｜30分

**定义**：用户在把任务交给 Agent 之前，能否主动构建包含必要要素的任务描述。

### 第一轮：自述习惯（16分）

> 「平时使用 AI 的时候，例如用 Claude Code 完成 coding 任务或者让 GPT 生成学术内容，一般会怎么跟他描述任务，prompt 会怎么写？请具体讲讲你的习惯。」

**探针（提到即得分）：**
- [ ] 提到会说明任务目标（任何"要做什么"的描述即可）：**6分**
- [ ] 提到会给 AI 设定角色或身份：**5分**
- [ ] 提到会提供背景信息或上下文（工具、引用文件、粘贴代码等）：**3分**
- [ ] 提到会让 AI 自检或配置验证，或要求 AI 列出需要人工核实的待确定项：**2分**

### 第二轮：缺陷 prompt 情景题（14分）

**场景说明：**
> 你打开了一个新的 Cursor/Claude Code 窗口，准备继续开发一个 MBTI 测试网站。项目目录大致如下：
>
> ```
> docs/
> src/
> scripts/
> ```
>
> 问卷页和访谈页已经完成，你现在的任务是实现报告页的得分展示模块。你写下了这条 prompt 发给 AI：
>
> 「帮我设计 MBTI 报告页的得分展示，要好看，风格简洁，用紫色系。」

**问题：**
> 你觉得这个 prompt 有什么问题？如果是你，你会怎么改？或者有什么建议？

**探针（提到即得分）：**
- [ ] 指出任务目标不完整（"描述太简单"或"没说清楚展示形式"即可）：**6分**
- [ ] 指出没有给 AI 设定角色：**3分**
- [ ] 指出应该让 Agent 了解当前项目已有内容/记忆文件：**2分**
- [ ] 指出没有要求自检或验证：**3分**

---

## Workflow（工作流）｜20分

**定义**：用户在处理复杂任务时，是否有"拆解-执行-验证-继续"的闭环工作范式。

**测量：一轮情景问题**

> 「你在处理一个复杂任务时，会怎么与 Agent 协作？如何保证结果质量？」

用户可用自己的真实经历回答，无追问。

**探针（提到即得分）：**
- [ ] 提到会分阶段推进，而不是一次性全部给 Agent：**8分**
- [ ] 提到步骤之间有顺序依赖：**2分**
- [ ] 提到会在中途局部验证/修复，确认当前步骤对了再继续：**6分**
- [ ] 提到 Agent 上下文有限，需要注意信息管理/开新对话：**4分**

---

## Repair（复原）｜20分

**定义**：Agent 出错或结果不符合预期时，用户是否有系统性的诊断和恢复思路。

**测量：一轮情景题**

> 「你让 Claude Code 帮你实现一个功能，它给了你一段代码，但运行之后结果不完全是你想要的。你会怎么做？」

**探针（提到即得分）：**
- [ ] 提到会定位具体哪部分结果不满意：**5分**
- [ ] 提到会判断是 prompt 问题还是 Agent 能力/上下文问题：**10分**
- [ ] 提到会把出错部分从整体任务中隔离出来单独处理：**5分**

## 评分系统

### 总分等级

| 等级 | 总分范围 |
|-----|---------|
| L1 新手 | 0 - 40 |
| L2 熟悉 | 41 - 70 |
| L3 专家 | 71 - 100 |

### 维度最低分门槛

达到总分且所有维度满足最低分，才能进入该等级。

| 维度 | 满分 | L2 最低 | L3 最低 |
|-----|-----|--------|--------|
| Route | 30 | 10 | 20 |
| Frame | 30 | 10 | 20 |
| Workflow | 20 | 5 | 12 |
| Repair | 20 | 5 | 12 |

> L1 无维度最低限制，总分 0-40 即可。

## 技术架构

### Agent 分工

- **Agent A（qwen-plus）**：按固定顺序提问，收集并记录用户原始回答
- **Agent B（qwen-max）**：接收原始回答 + 探针清单，判断探针触发情况，生成结构化评分，生成报告

### 访谈顺序与时长

| 轮次 | 内容 | 预计时长 |
|-----|-----|---------|
| 1 | Route 开放问题 | 30秒 |
| 2 | Frame 自述习惯 | 60秒 |
| 3 | Frame 缺陷 prompt | 90秒 |
| 4 | Workflow 情景问题 | 45秒 |
| 5 | Repair 情景题 | 45秒 |
| | **合计** | **~5分钟** |

### Agent B 输入格式

```json
{
  "route_answer": "用户原始回答",
  "frame_answer_1": "用户原始回答",
  "frame_answer_2": "用户原始回答",
  "workflow_answer": "用户原始回答",
  "repair_answer": "用户原始回答"
}
```

### 探针清单（Agent B 判断依据）

**Route（30分）**
- 提到 Agent 可调用外部工具/MCP：10分
- 提到 Agent 有记忆/状态管理：10分
- 提到需要人工判断/不直接使用输出：10分

**Frame 第一轮（16分）**
- 提到任务目标：6分
- 提到角色/身份设定：5分
- 提到背景信息/上下文：3分
- 提到自检/验证：2分

**Frame 第二轮（14分）**
- 指出任务目标不完整：6分
- 指出没有角色设定：3分
- 指出应让 Agent 了解已有项目内容：2分
- 指出没有自检/验证要求：3分

**Workflow（20分）**
- 提到分阶段推进：8分
- 提到步骤顺序依赖：2分
- 提到中途局部验证/修复：6分
- 提到上下文限制意识：4分

**Repair（20分）**
- 提到定位具体问题：5分
- 提到归因判断（prompt问题 vs Agent能力/上下文）：10分
- 提到问题隔离：5分

### 模型配置

- Agent A：`QWEN_PLUS_MODEL`（对话访谈）
- Agent B：`QWEN_MAX_MODEL`（评分 + 报告生成）
- Phase 1 报告生成同步升级为 `QWEN_MAX_MODEL`
