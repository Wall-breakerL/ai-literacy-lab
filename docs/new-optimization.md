# 当前自检优化记录

记录日期：2026-05-15

## 背景

S8-S10 完成后，Owner 又做了一轮完整自检。前面两轮优化方案先归档，不再作为当前实施依据；本文件只保留最后这一轮自检发现的问题，并按 PR 拆分处理。

## PRA：先做的明显 UI / 内容问题

### A1. 中途反馈页的场景经历也需要选项

- 发生位置：中途反馈页，`我想指定第二轮方向` 展开区。
- 现象：这里只能手填，和开始资料页的场景词条不一致；不喜欢主观题的用户仍然会有退出感。
- 目标：像开始资料页一样提供可点击的场景词条，同时保留自定义输入。
- 说明：词条要告诉用户用途，即这些词会作为第二轮题目的方向提示。

### A2. 海报页左右两侧双线条显得拥挤

- 发生位置：报告海报页，海报卡片左右两侧。
- 现象：左右边缘有外框线和内侧竖线，视觉上像双线条，手机端尤其显挤。
- 目标：去掉内侧竖线，只保留海报本身的边框和内容结构。

### A3. 报告切换主题没有明显变色

- 发生位置：报告故事页 / 海报页所在的报告体验外壳。
- 现象：切换 Light / Dark 后，报告页面主体仍然像硬编码深色，没有跟随全局主题变量。
- 目标：报告故事页的页面背景、slide 卡片、基础文字和导航控件跟随主题变化。
- 边界：海报图片本身可以保持固定深色视觉，因为它是生成 / 分享用的海报资产。

### A4. Docs 只保留当前轮自检问题

- 发生位置：`docs/new-optimization.md`。
- 现象：旧的两轮计划和当前自检问题混在一起，后续讨论容易误判优先级。
- 目标：清理为当前自检轮问题，并保留 PR 拆分。

## PRB：下一轮仔细定位的问题

### B1. 电脑端分享图片 / 拷贝仍然出现两张图片

- 发生位置：报告海报页，电脑端点击分享按钮。
- 现象：一次操作仍然可能生成两张图片或触发两次下载 / 拷贝。
- 需要定位：是 `handleSharePoster` 的降级分支重复触发，还是浏览器分享 / 下载能力判断走了两条路径。

### B2. 手机端分享按钮失败

- 发生位置：报告海报页，手机端点击分享按钮。
- 现象：点击后失败，用户拿不到可用的分享结果。
- 需要定位：`html2canvas` 生成失败、`navigator.canShare` 不支持文件、移动端内存 / 字体 / SVG 兼容问题，还是失败提示和降级策略不够明确。

## 当前处理顺序

1. 先完成 PRA，并自检后合并。
2. 再单独打开 PRB，重点分析分享链路，避免把 UI 调整和浏览器分享兼容性混在一个 PR 里。
3. PRB 完成后，再单独排查第二轮问卷容易进入 fallback 的问题。
