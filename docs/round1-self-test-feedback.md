## 合并后的核心反馈

### D1. 主观输入门槛偏高，影响继续完成率

- 来源反馈：012、013
- 发生位置：信息收集页、中途反馈页
- 核心反馈：用户不喜欢一开始或中途被要求写主观题；看到必填文本框容易直接退出。
- 问题本质：当前流程把”高质量补充信息”和”完成主流程”绑定得太紧，增加了低意愿用户的作答阻力。
- **确定方案：**
  - intake 页”具体 AI 使用经历”改成中等颗粒度场景标签（如”写代码调试”、”写需求文档”、”学习新知”、”数据分析”、”头脑风暴”、”做PPT报告”等），按已选职业展示 4–5 个对应场景，支持最多选 2 个，最后加”自己补充场景”选项，点开后才出现自定义填写框。
  - 场景词条区必须说明用途：这些词条会用来生成第一轮问卷，让题目更贴近用户真实使用方式，避免用户不知道为什么要选。
  - 中途反馈页把”具体题目反馈”和”第二轮场景方向”折叠成两个展开按钮，默认不展示；展开区域内加一句提示说明，告诉用户填了会影响第二轮个性化（例如”告诉我你想聚焦的场景，第二轮题目会更贴近你的实际工作”）。

### D2. 问卷作答效率和量表表达需要优化

- 来源反馈：001、006、016
- 发生位置：问卷作答页
- 核心反馈：点击 0-5 分数后还要再点”下一题”，效率偏低；”上一题”字体比”下一题”更亮，主次视觉不对；量表词”偶尔会 / 经常会 / 通常会”强弱顺序容易引起歧义。
- **确定方案：**
  - 非末题点击选项后自动跳转下一题，加约 300ms 缓冲让用户看清选择再跳；末题保留明确的”提交问卷”确认，不自动跳转；”不了解 / 没想好”保留手动确认，避免误触。
  - “上一题”按钮字体颜色从 `text-light-gray` 调暗至 `text-dim-gray`，降低视觉权重，与”下一题”主次拉开差距。
  - 量表从行为频率改为 6 级同意度，文案定为：**完全不同意 / 不同意 / 有点不同意 / 有点同意 / 同意 / 完全同意**；保留 0–5 数字按钮，并同步报告答题明细里的分数标签。

### D3. 问卷题目质量：两轮重复、题面偏向性强

- 来源反馈：008、015
- 发生位置：第一轮问卷、第二轮问卷
- 核心反馈：第一轮和第二轮差异不明显，甚至出现近似重复题；部分题目太容易看出测量方向，用户能提前推断结果，导致报告缺少冲击力。
- **确定方案：本轮只做保守重复题保险丝。**
  - 相似度阈值放高到 `0.98`，只拦截几乎完全一样的题；只要不是高度重复，优先放行。
  - 首次发现重复时最多让模型重试一次；重试后仍重复则记录 warning 并放行，不切 fallback，避免流量升高时 fallback 放大问题。
  - 题面隐蔽性、结果冲击力和深层题目改写暂缓，留到后续单独设计。

### D4. 报告可信度：结构、标签和算法解释不够清楚

- 来源反馈：009、010、011
- 发生位置：用户风格总览、人格类型判定、协作风格画像
- 核心反馈：报告里有些模块职责不清，风格类型可能看起来自相矛盾；全选四得到”待观察”不符合用户直觉；协作风格画像第二项可能被理解成把用户说反。
- **确定方案：**
  - 在完整报告”答题与计分明细”区块的 `tendencyLabel` 旁边，如果值为”待观察”则增加一句小注说明：「正反向题得分相近，该维度信号较弱」，条件渲染，改动最小。
  - 报告第3页”你的协作风格画像”（slide）中，移除 `styleProfile.comparison` 对比区块，只保留用户自己的风格分析（`styleProfile.uniqueness` 或行为描述），字体适当放大，排版重新做好看。完整报告展开区的 comparison 区块同步移除。

### D5. 报告资料页偏暗，需要阅读主题或局部提亮

- 来源反馈：002
- 发生位置：报告开头资料页 / 画像资料页
- 核心反馈：资料页里的选项或标签偏暗，不同用户对亮度偏好不同。
- **确定方案：全局 Light / Dark 主题切换，本轮实施。**
  - 范围：整个应用统一切换，intake、问卷页、中途反馈页、报告页全部响应。
  - 实现：在 `globals.css` 定义两套 CSS 变量，dark 是现有值，light 是亮色对应值；selector 用 `:root[data-theme="light"]`；切换按钮放在全局 layout 右上角，用户选择后存入 `localStorage`，刷新保持。
  - 颜色分类：背景、卡片、文字、边框等结构色参与切换；Raycast Red、人格主题色、光谱渐变色等品牌色两套主题均保持不变。
  - 实施边界：主题切换覆盖主流程结构色和报告展开内容；海报视觉、人格主题色、Raycast 品牌色保持稳定，不跟随 Light mode 重新配色。

### D6. 海报页移动端展示和操作入口不舒服

- 来源反馈：003、007、014
- 发生位置：报告海报页，手机端优先
- 核心反馈：海报页底部按钮过大；反馈和返回首页入口藏得太深；长文案或两行标题会让海报底端需要下滑才能看到。
- **确定方案：**
  - 海报容器改用 `100svh` 自适应高度，操作区高度固定，海报 `max-h = 100svh - 操作区高度`，保证任何文案长度下首屏完整可见。
  - 底部改为三个轻量并排小按钮：**分享 / 保存图片 · 查看完整报告 · 反馈**。
  - 返回首页改到左上角小箭头，不占底部空间。
  - 扩展 `/debug-report` 页面，支持选择 16 种人格直接生成对应报告，供调试验证海报首屏适配效果；验收标准是 16 个人格的海报在移动端首屏都完整可见。

### D7. 海报结果缺少保存和分享能力

- 来源反馈：004
- 发生位置：报告海报页
- 核心反馈：用户离开网站后海报不保留，不方便保存或分享。
- **确定方案：**
  - 用 `html2canvas` 或 `dom-to-image` 把海报 DOM 生成图片。
  - 优先调用 Web Share API（移动端弹系统分享面板）；浏览器不支持时降级为直接下载图片。
  - 入口是 D6 底部三个按钮中的”分享 / 保存图片”按钮。
  - 分享链接本轮不做，留后续迭代。

### D8. 首页特点卡片的交互暗示不清

- 来源反馈：005
- 发生位置：开始界面三个特点卡片（”3 分钟轻量访谈”、”16 题定制问卷”、”个性化协作建议”）
- 核心反馈：三个特点卡片看起来像可以选中，但实际只有 hover 发光，没有点击结果。
- **确定方案：弱化为纯展示。** 移除 `whileHover` 卡片上浮动效果、移除内部 radial gradient 发光层、移除图标 `group-hover:scale-110` 放大效果，让卡片变成静态展示块，不产生可点击预期。

## 建议讨论顺序

1. D1 主观输入门槛：直接影响完成率。
2. D3 问卷题目质量：直接影响测评可信度和结果惊喜。
3. D4 报告可信度：决定用户是否相信结果。
4. D6 海报页移动端展示：是最终结果的第一印象。
5. D2 问卷作答效率和量表：中等成本、体验收益明确。
6. D7 海报保存分享：传播价值高，但要控制实现范围。
7. D5 报告亮度主题：可读性问题，需决定做局部还是系统主题。
8. D8 首页卡片交互：低到中优先级，适合顺手修。

## 给 Codex 的实施说明

1. D3 当前开发分支只做保守重复题保险丝，不做题面隐蔽性重写，不改变问卷维度和评分逻辑。
2. D5 当前开发分支已选择全局主题切换方案；实现范围聚焦结构色、文字、边框和报告展开内容，海报视觉不跟随主题改色。
3. D1/D2/D8 已由 PR #4 实施，只改前端交互和文案，不改问卷评分逻辑、报告生成 API 或 session 数据结构。
4. D4 当前开发分支不改评分算法，只调整报告生成结构和前端解释：新报告不再要求生成 comparison，旧报告即使带 comparison 也不再展示。
5. D6 的验收标准：16 个人格在移动端首屏均完整可见，需用扩展后的 /debug-report 逐一验证。PR #3 已先验证 CEAL 和最长 Golden Line 的 IFAL 移动端首屏适配。
6. D7 使用 `html2canvas` 或 `dom-to-image`，选择更轻量、兼容性更好的方案即可。PR #3 已采用 `html2canvas`，支持 Web Share，失败时降级下载 PNG。
