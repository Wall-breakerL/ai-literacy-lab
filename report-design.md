# AI-MBTI 报告页设计规格文档

> 版本：v1.0 · 整理自设计对话 · 2026-05-03

---

## 一、已确认的核心决策

| 决策项 | 方案 | 说明 |
|--------|------|------|
| 主导分数显示 | **不显示** | 用视觉（指示器大小/亮度）暗示极值，不用数字强调 |
| 幻灯片导航 | **手动点击** | 用户节奏自主，不自动播放 |
| 海报比例 | **9:16 竖版** | 优先适配手机截图分享 |
| 整体流程 | **加载 → 幻灯片(4页) → 海报 → [可选]完整报告** | 线性递进，不跳环节 |

## 二、色彩系统

每个维度用一对颜色表示光谱的两端，整条渐变条呈现连续过渡（非硬分段）。

| 维度 | 低分端（字母·名称） | 颜色 | 高分端（字母·名称） | 颜色 | 语义对比 |
|------|------------------|------|------------------|------|---------|
| Relation 关系定位 | I · 工具型 | `#2563eb` 深蓝 | C · 伙伴型 | `#f97316` 橙红 | 冷静 → 温暖 |
| Workflow 工作流程 | F · 框架型 | `#4f46e5` 靛青 | E · 探索型 | `#14b8a6` 青绿 | 结构 → 流动 |
| Epistemic 认知态度 | A · 审计型 | `#64748b` 石墨灰 | T · 信任型 | `#fbbf24` 金黄 | 谨慎 → 信任 |
| RepairScope 修复范围 | G · 全局型 | `#8b5cf6` 紫罗兰 | L · 局部型 | `#10b981` 翠绿 | 宏观 → 微观 |

**配色原则：**
- 四个维度的主色（蓝、靛、灰、紫）在色轮上均匀分布，并列时不互相干扰
- 高分端（C/E/T/L）统一偏暖色/亮色，视觉上带有积极联想
- 渐变条实现方式：整条用 `linear-gradient(起始色, 结束色)` 绘制，通过指示器位置表达分数，不做硬切割

**无障碍说明：**
- 石墨灰 `#64748b` 在白底上对比度约 4.6:1，建议配白色文字时使用深色背景衬托
- 金黄 `#fbbf24` 建议文字色用 `#78350f`（深棕），避免黄底白字对比度不足

## 三、光谱条组件规格

```
  G 全局型                              L 局部型
  ●━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━●
  [紫罗兰渐变 ←——————————————————→ 翠绿渐变]
                                  ◆ 指示器（位置 = 分数%）
```

| 元素 | 规格 |
|------|------|
| 渐变条高度 | 8px，圆角 4px |
| 两端锚点（●） | 直径 10px，颜色与对应端一致 |
| 指示器（◆） | 16×16px 菱形，白色描边 2px，drop-shadow |
| 指示器动效 | 进场时从中点滑动到目标位置（duration 800ms，easing ease-out） |
| 极值强调规则 | 若分数 ≥ 75 或 ≤ 25（距中位偏差 ≥ 25），指示器增大至 20px 并加呼吸动画 |
| Hover 交互 | 显示 tooltip："你在 X 型这一端（分数 N）" |
| 分数显示 | 指示器右侧 4px 处，字号 12px，色同指示器端颜色，幻灯片中显示，海报中隐藏 |

## 四、幻灯片结构（共 4 页）

### 导航机制
- 底部圆点进度指示器（4个点，当前页高亮放大）
- 左右箭头按钮，首页隐藏左箭头，末页箭头变为"查看海报"
- 支持左右滑动手势（移动端）
- 键盘支持：← → 方向键翻页

### 切换动效
- 水平滑动切换（向左推出/向右推入），非淡入淡出
- duration 300ms，easing cubic-bezier(0.4, 0, 0.2, 1)

### 第 1 页：人物卡片（Who）

**目标：** 让用户记住"我是谁"

**布局：**
```
┌─────────────────────────────────────┐
│                                     │
│         [PersonalityAvatar]         │
│              168×168px              │
│                                     │
│          CETL · 框架探索者           │
│      "先定框架，再让AI自由发挥"       │
│                                     │
│  [探索型] [审计型] [局部迭代] [伙伴] │
│                                     │
└─────────────────────────────────────┘
```

**内容规则：**
- 类型名称（如"框架探索者"）：字号 22px，加粗，主题色
- Tagline：字号 14px，灰色，引号包裹
- Tag 徽章：圆角胶囊，背景色取对应维度高分端颜色，透明度 15%，文字色为该颜色全量版本
- Tag 只显示用户偏向明显的那侧（分数 ≥ 60 才展示）

**内容规则：**
- 序号圆圈：直径 24px，渐变背景（取当前用户最强维度的颜色对），白色数字
- 每条洞察标题：字号 14px，加粗
- 正文：字号 13px，行高 1.6，灰色 `#475569`
- 每条之间分割线：`1px solid #f1f5f9`

## 五、海报规格（第 5 屏）

**比例：** 9:16 竖版，建议画布尺寸 1080×1920px（或等比缩放至 390×693px 用于预览）

**目标：** 截图即可分享，一眼看懂

**布局分区：**

```
┌─────────────────────────────────┐  ← 上边距 60px
│                                 │
│  [Avatar 120px]  CETL           │  ← 头像 + 类型标识区
│                  框架探索者      │
│                  "先定框架..."   │
│                                 │
│  ─────── 分割线 ───────          │
│                                 │
│  [四维光谱条，缩小版，无分数]    │  ← 光谱区（无数字，只看位置）
│  I ←━━━━●━━━━━━━━━━━━━→ C    │
│  F ←━●━━━━━━━━━━━━━━━━→ E    │
│  A ←━━━━━━━━━━━━●━━━━→ T    │
│  G ←━━━━━━━━━━━━━━━━●━→ L   │
│                                 │
│  ─────── 分割线 ───────          │
│                                 │
│  核心模式   框架型 + 局部迭代    │  ← 摘要区（3行关键信息）
│  适合场景   复杂项目 · 技术方案  │
```

**海报专属规则：**
- 背景：深色渐变（`#0f172a` → `#1e293b`），文字用白色系
- 光谱条高度缩至 4px，两端锚点缩至 6px，指示器缩至 12px（无分数标注）
- 分割线：`1px solid rgba(255,255,255,0.12)`
- 下载/分享按钮：固定在海报预览下方（不包含在画布内），点击触发长按保存提示或调用系统分享

## 六、整体流程与页面跳转

```
加载动画（1.5-2s）
    ↓
第1页：人物卡片
    ↓ [手动点击下一页]
第2页：四维光谱
    ↓ [手动点击下一页]
第3页：核心洞察
    ↓ [手动点击下一页]
第4页：行动指南
    ↓ [点击"查看海报"]
第5屏：海报
    ↓ [可选，底部文字链接]
完整报告（滚动长页）
```

**各跳转节点说明：**
- 幻灯片 → 海报：末页箭头按钮文字改为"查看我的海报 →"，带有微动效引导
- 海报 → 完整报告：海报页底部附一行低调文字链接"查看完整维度解读 ↓"，不强调
- 完整报告不作为默认终点，降低信息过载感

## 七、加载动画规格

动画分三阶段，总时长约 1.8s：

| 阶段 | 内容 | 时长 |
|------|------|------|
| 1 | 四条光谱从中点向两端展开（同时播放，带颜色） | 0.6s |
| 2 | 类型代码字母逐个显现（如 C → E → T → L） | 0.5s |
| 3 | 类型名称淡入（如"框架探索者"） | 0.4s |
| 过渡 | 整体向上移出，第1页滑入 | 0.3s |

## 八、优化补充建议

以下是对原始设计方案的补充优化，可按优先级分期实现：

### 优先级 P0（核心体验，必须有）

1. **指示器极值强调**：分数偏离中位 ≥ 25 的维度，指示器自动放大并加呼吸动效，无需文字说明即可让用户感知"哪里最突出"
2. **Prompt 一键复制**：第4页复制按钮必须可用，这是用户最可能立刻行动的出口
3. **海报长按保存**：移动端需监听 `contextmenu` 或提示"长按图片保存"，确保分享路径顺畅

### 优先级 P1（体验提升，建议有）

4. **幻灯片滑动手势**：移动端支持左右滑动翻页，降低操作成本
5. **进场动效错开（stagger）**：第2页4条光谱错开 100ms 依次入场，避免所有元素同时运动造成视觉混乱
6. **Tag 只显示偏向明显维度**：分数 ≥ 60 才生成对应 Tag，避免全量展示导致 Tag 失去意义

### 优先级 P2（锦上添花，有时间再做）

7. **海报背景主题**：提供 2 套海报背景（深色极简 / 浅色清爽），用户可切换后再保存
8. **QR Code 可配置**：若暂无二维码服务，海报底部可改为纯文字 URL，后续接入再替换
9. **完整报告锚点导航**：若完整报告内容较长，顶部加固定导航栏（4个维度名可点击跳转）
